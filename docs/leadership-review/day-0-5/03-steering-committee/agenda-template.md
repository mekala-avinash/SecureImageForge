# Steering Committee — Standing Agenda Template

> Cadence: biweekly Fridays 10:00–11:00 · Chair: VP Platform · Notes: Program PM

## Pre-read (sent 24h prior)
- Current burndown (Jira / Linear)
- Risk register diff
- Budget actuals vs plan
- Open decisions ledger

## Agenda (60 min)

### 0:00–0:05 — Roll call & approve prior minutes
- Quorum check (5/7 voting).
- Approve prior meeting minutes (or capture corrections).

### 0:05–0:15 — Program status (Program PM)
- RAG status by workstream:
  - Hardened runtimes
  - GitOps + Argo
  - Security / supply chain
  - Observability
  - Agents
  - Compliance
  - Multi-region / cloud
- Milestones reached this cycle.
- Milestones at risk this cycle (with named owner + ask).

### 0:15–0:25 — Decisions ledger (Chair)
- Pending decisions (≤ 5).
- New decisions requested today.
- Vote where required; capture dissents.

### 0:25–0:35 — Risk register (Chair)
- New risks (≥ MEDIUM).
- Risk movements (likelihood/impact changes).
- Mitigations status.
- Risk acceptances proposed.

### 0:35–0:45 — Budget & FTE (CFO + VP HR delegate)
- Burn vs plan (per phase, per workstream).
- Hiring funnel (open reqs, interviewed, accepted, started).
- Contractor utilization vs cap.

### 0:45–0:55 — Compliance & security (CISO + Head of Compliance)
- Audit calendar status.
- Findings remediation status.
- Policy-as-code coverage trend.

### 0:55–1:00 — Wrap
- Recap decisions made.
- Action items + owners + due dates.
- Next meeting reminders.

## Phase-gate variant (when scheduled)

Extend to 2 hours. Same agenda first 60 min, then:

- **Hour 2 — Phase-gate review** (gate-keeper drives)
  - Walk through exit criteria checklist.
  - Independent verification evidence (artifacts, dashboards, audit logs).
  - Risk re-baseline.
  - Vote: GO / CONDITIONAL GO / NO-GO.
  - If NO-GO: 2-week corrective window agreed; re-review scheduled.
